★ 반복 실험 종합 인사이트

① 클래스 가중치 보정의 실질 효과

보정 후 Macro F1·AUROC는 소수 클래스(RPI 4~5) 예측이 강화되어 상승하는 경향이 있으며, 반면 Micro F1·AUROC는 다수 클래스(RPI 1~2) 정확도가 일부 희생되어 소폭 하락할 수 있습니다. 이 트레이드오프는 비즈니스 목적에 따라 선택해야 합니다.
→ 위험 고객(RPI 4~5) 조기 탐지가 목적이라면 보정 적용, 전체 고객 관계 모니터링이 목적이라면 미보정이 적합합니다.

② Random Forest의 높은 분산(Std)이 의미하는 것

미보정 조건에서 Random Forest의 Macro F1 Std가 타 모델 대비 크게 나타납니다. 이는 배깅(bagging)의 무작위 샘플링이 클래스 불균형 데이터에서 소수 클래스 포함 비율을 seed별로 다르게 만들기 때문입니다. 보정 적용 후 Std가 줄어드는지 확인하는 것이 중요합니다.

③ Logistic Regression의 일관된 우위

두 조건 모두에서 Logistic Regression은 Std=0.0000으로 완전 안정적이며 Macro F1·AUROC 모두 최상위권입니다. PCA로 변환된 공간에서 RPI와의 관계가 선형 분리 가능 구조를 유지하기 때문입니다.
→ 운영 모델의 1순위 후보: 안정성·성능·속도 세 요소를 모두 충족합니다.

① 모델별 안정성 비교

5회 반복 실험에서 각 모델의 Macro F1 Std를 기준으로 안정성을 평가합니다.
Logistic Regression과 Random Forest는 std가 매우 작아 재현성이 높습니다. LightGBM은 부트스트랩 샘플링과 특성 서브샘플링으로 인해 시드별 변동이 상대적으로 큽니다. Decision Tree는 결정론적 분기 특성상 시드 변화의 영향이 제한적입니다.

② 통계 검정 결과가 시사하는 것

Friedman test 결과를 기반으로 모델 간 차이의 통계적 유의성을 확인합니다. 유의한 지표에서는 Wilcoxon 사후 검정을 통해 어떤 모델 쌍이 실제로 다른지 파악합니다. Bonferroni 보정을 적용하여 다중 비교 오류(Family-wise error)를 통제합니다.

관점내용
Decision Tree vs 앙상블 Decision Tree는 단일 분기 구조 특성상 PCA 공간에서도 과적합 또는 과소적합 위험이 있음. Random Forest·LightGBM은 다수 트리 앙상블로 분산을 낮춰 성능 개선이 기대됨.
Macro vs Micro 지표 Macro 지표는 소수 클래스(RPI 2 — 8건)에 민감. 클래스 불균형이 존재하는 Test 셋에서 Macro < Micro 패턴이 나타나면 소수 클래스 예측 난이도가 높다는 신호.
PCA 적용 효과 51개 원본 feature를 40개 PC로 압축해 노이즈·다중공선성 감소. 특히 Decision Tree 계열에서 과적합 억제 효과가 크게 나타남.
학습 효율 PCA로 입력 차원이 줄어 LightGBM·RF 모두 빠른 학습 속도 유지. 운영 환경에서는 학습시간과 성능 간 트레이드오프를 고려해 모델을 선택.

★ 종합 인사이트 & 핵심 발견

① LightGBM의 Macro F1(0.9423) 이상 현상 분석

LightGBM은 Macro AUROC 0.9999(최고)이면서 동시에 Macro F1 0.9423(최저)이라는 역설적 결과를 보입니다.
원인: Test 셋에서 RPI 2 클래스가 단 8건으로 극소수입니다. LightGBM은 확률 추정(AUROC)은 우수하지만, 의사결정 경계에서 RPI 2를 인접 클래스(RPI 3)로 잘못 분류하는 경향이 있습니다. 이로 인해 RPI 2의 F1이 0에 가까워지고 Macro 평균을 크게 끌어내립니다.
→ 클래스 가중치(class_weight='balanced') 적용 또는 임계값 최적화로 개선 가능합니다.

② Random Forest: 균형 잡힌 최선의 모델

Random Forest는 Macro F1 0.9947, Micro F1 0.9933, Micro AUROC 0.9998로 4개 모델 중 가장 균형 잡힌 성능을 보입니다. 학습시간 0.39s로 LightGBM(2.254s) 대비 약 6배 빠르면서도 전 지표에서 앞서거나 동등합니다.
Decision Tree 대비 Macro F1 +0.0053, Micro AUROC +0.0087 향상.
→ 운영 환경 배포 1순위 모델로 권장됩니다. 특히 소수 클래스 예측 안정성이 필요한 경우에 적합합니다.

③ Logistic Regression: 비용 대비 효율 최고

학습시간 0.042s로 LightGBM 대비 54배 빠르며, Macro F1 0.9922로 Random Forest와 불과 0.0025 차이입니다. 이는 PCA로 변환된 데이터 공간에서 RPI와 PC들 사이의 관계가 선형 분리 가능함을 증명합니다.
→ 실시간 예측 API, 배치 처리 시스템 등 속도가 중요한 환경에서 최우선 선택지입니다.

④ Macro와 Micro 지표 차이가 말해주는 것

모든 모델에서 Macro F1 ≤ Micro F1 패턴이 관찰됩니다. Test 셋 클래스 분포(RPI 2: 8건, RPI 3~5: ~100건씩)의 불균형이 원인입니다.
LightGBM의 경우 격차가 가장 크며(Macro 0.9423 vs Micro 0.9833 = -0.0410), Random Forest는 격차가 가장 작습니다(0.9947 vs 0.9933 = -0.0014).
→ 실무에서 소수 등급(RPI 1~2, 관계 우수 고객사)을 중요하게 여긴다면 Macro 지표 중심으로 모델을 선택해야 합니다. 반면 전체 정확도(Micro)가 우선이라면 어떤 모델이든 우수합니다.

② Ordinal Classification 실험 RPI 1→5는 서수형(순서 있음) 데이터 — 일반 다중분류는 순서 무시 OrdinalClassifier (mord / sklearn-ordinal) 또는 누적 로짓 모델 비교 RPI 2→4 혼동 감소, 인접 클래스 오분류 페널티 반영
③ Calibration 검증 (확률 보정) LightGBM Macro AUROC 0.9999 — 출력 확률이 실제 비율과 일치하는지 미검증 Reliability Diagram + ECE(Expected Calibration Error) 측정, 필요시 Isotonic / Platt Scaling 확률 기반 위험 순위화 신뢰도 확보 + 운영 의사결정 품질 향상

🟡 Mid Priority — 1~2개월

과제배경방법기대 효과
④ 5-class 완전 평가 셋 구축 현재 Test 셋에 RPI 1 클래스 없음 → 4-class 기준 지표만 산출 RPI 1 포함 균형 Test 셋 별도 구성 또는 k-fold 전체 클래스 보장 분할 실제 운영 환경 반영 5-class 완전 성능 평가 가능
⑤ 모델 앙상블 (Stacking / Voting) LR(안정성) + LGB(AUROC) + RF(다양성) 세 모델의 강점이 상이 Soft Voting 또는 LR 메타 분류기 기반 Stacking 단일 모델 대비 Macro F1 추가 향상 가능성
⑥ 하이퍼파라미터 최적화 현재 기본값(n_estimators=200, C=1.0 등) — 최적값 미탐색 Optuna Bayesian Optimization + Stratified K-Fold CV (macro_auroc 기준) 각 모델 1~2% 추가 성능 향상 기대
⑦ SMOTE vs class_weight 정량 비교 현재는 class_weight 보정만 실험 — SMOTE 오버샘플링 미비교 SMOTE / ADASYN / BorderlineSMOTE 각각 5-seed 실험 후 현재 결과와 비교표 작성 클래스 불균형 처리 전략별 정량적 효과 문서화

🟢 Low Priority / 장기

과제배경방법기대 효과
⑧ 온라인 학습 / 점진적 업데이트 연간 신규 데이터 누적 — 매년 재학습 필요 Incremental Learning (SGD / River 라이브러리) 또는 연간 재학습 파이프라인 자동화 데이터 드리프트 대응 + 운영 지속 가능성 확보
⑨ 모델 공정성 (Fairness) 검증 고객사·제품군별 예측 성능 차이 가능성 미검토 서브그룹(product/area)별 Macro F1 분해 분석 특정 그룹 불이익 탐지 + 공정한 서비스 품질 보장
