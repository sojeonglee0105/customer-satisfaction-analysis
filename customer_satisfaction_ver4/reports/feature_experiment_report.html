Exp 2  Feature Selection 선택 근거

선택 기준 (OR 조건 — 둘 중 하나라도 충족하면 선택)
  • Mutual Information (MI) ≥ 평균 MI (0.7164)  — 비선형 관계 포함 target과의 정보량 기준. MI가 평균 이상인 변수는 클래스 구분에 실질적 기여를 하는 것으로 판단.
  • Pearson |r| ≥ 0.15  — 선형 상관이 일정 수준 이상인 변수 보강. MI만으로 포착 못하는 선형 신호를 추가로 포함.

→ MI ≥ 0.7164를 충족하는 변수 21개, Pearson |r| ≥ 0.15를 충족하는 변수 48개, 합집합으로 최종 48개 변수를 선택.

인사이트 요약

관점내용
Feature 수 영향 전체 51개 feature 중 상당수가 중복 정보를 담고 있을 가능성이 높음 (CSI·CCI 하위 항목 구조상 동일 영역의 res/core/comm/total이 공존). Feature Selection·PCA로 압축 시 노이즈 감소 효과 기대.
Feature Selection 의의 MI + Pearson 조합으로 비선형·선형 신호를 모두 고려. 선택된 48개 변수는 원본 대비 약 94% 수준으로 압축되면서도 핵심 정보를 보존.
PCA 의의 25개 PC로 원본 51개 변수의 95.3% 분산을 설명. 모델 간 성능 변화 폭이 크다면 원본 feature 공간의 다중공선성 문제를 간접 확인 가능.
주의 사항 Test 셋에 RPI 1 클래스가 없어 macro 지표가 4-class 기준으로 산출됨. 실제 서비스에서는 전체 5-class 평가가 필요.

🏭 영역별 도메인 관점 Feature Engineering 재해석

변수명 접두사: t1=신기술 / t2=개발 / c=Cost / d=공급 / q1=품질 / q2=서비스

PCA가 최고 성능인 이유 — 도메인 관점
t1(신기술)·t2(개발)·c(Cost)·q1(품질)·q2(서비스)·d(공급)의 res/core/comm 세부 항목이 동일 영역 내에서 r≥0.9 다중공선성을 형성합니다. PCA는 6개 영역 × 2지수(CSI/CCI) = 12개 잠재 축으로 자연스럽게 압축되므로 성능이 가장 우수합니다.
Feature Selection 한계 — 도메인 관점
MI + Pearson 기준 Feature Selection이 48개를 선택했지만 영역별 세부항목(res/core/comm)을 개별로 다루어 동일 영역 내 중복 정보를 제거하지 못합니다. 도메인 지식 기반으로 "_total만 사용"하는 전략이 더 효율적일 수 있습니다.
권장: 영역 총점 피처셋 (12변수)
t1/t2/c/d/q1/q2 각 영역의 csi_total + cci_total = 12개 총점 변수만 사용. 영역 내 다중공선성 완전 해소 + 비즈니스 해석력 극대화. PCA 없이도 Logistic Regression이 높은 성능을 낼 것으로 예측됩니다.
다음 실험: 영역별 중요도 개별 분석
신기술(t1) vs Cost(c) vs 품질(q1) 영역별로 단독 모델 성능 측정 → "어떤 영역만으로도 RPI를 예측할 수 있는가?" 검증. 영역별 독립 예측 모델로 세분화된 CS 전략 수립 가능합니다.

🔬 추가 과제 제안 — Feature Engineering 실험 후속 로드맵

현재 실험(All / Feature Selection / PCA 3가지 전략)에서 확인된 결과를 토대로 성능을 더 끌어올리거나 해석력을 높일 수 있는 후속 Feature Engineering 과제를 제안합니다.

🔴 High Priority

과제현 실험 근거제안 방법기대 효과
① _total 12변수 단독 실험 Feature Selection이 48개 선택 → 사실상 전체와 유사, 세부항목 중복 CSI total 6 + CCI total 6 = 12변수만으로 동일 모델 재실험 변수 수 76% 감소 + 성능 유지 여부 확인 → 경량 운영 모델 근거
② PCA 컴포넌트 수 민감도 분석 95% 분산 기준 25 PC → 최적값 미검증 n_components = 5/10/15/20/25/30/95% 분산 별 성능 곡선 그리기 최소 PC 수 대비 성능 trade-off 확인 → 운영 복잡도 최소화
③ Incremental Feature Addition 실험 어떤 변수가 추가될 때 성능이 뛰는지 미확인 앙상블 순위 1위부터 순차 추가하며 Macro AUROC 변화 추적 최소 필요 변수 집합(Elbow Point) 식별 + 설문 문항 축소 근거

🟡 Mid Priority

과제현 실험 근거제안 방법기대 효과
④ PCA 대비 UMAP / Kernel PCA 비교 PCA(선형 압축)가 최고 성능 → 비선형 압축 미탐색 UMAP(n_components=10~20) / Kernel PCA(rbf) 동일 모델 비교 비선형 구조 존재 시 추가 성능 향상 가능
⑤ Domain-Knowledge 파생 변수 실험 CCI − CSI 차이(신뢰 대비 만족 갭) 등 비즈니스 의미 변수 미시도 CCI_total − CSI_total, 최근 분기 대비 변화율(Δ) 등 파생 변수 추가 원본보다 압축된 도메인 특화 시그널 발굴
⑥ Target Encoding (고객사 레이블) client 변수가 앙상블 중요도 Rank 7 — 단순 Label Encoding 한계 client별 RPI 평균 기반 Target Encoding + Leave-One-Out 교차검증 고객사 히스토리 정보를 피처에 반영, Macro F1 향상 기대
